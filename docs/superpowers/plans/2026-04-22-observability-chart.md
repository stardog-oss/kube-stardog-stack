# Observability Chart Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create `charts/observability` — a new subchart that deploys Prometheus, Grafana, Loki, and Alloy as an integrated monitoring stack, pre-configured to collect Stardog metrics and logs.

**Architecture:** A Helm composition chart with three upstream dependencies (`kube-prometheus-stack`, `loki`, `alloy`), each configured with `fullnameOverride` for predictable service names so Alloy can reference Loki at a static URL (`http://loki:3100`). The chart adds Stardog-specific templates (ServiceMonitor, PrometheusRule, Grafana dashboard ConfigMap) on top of the upstream stack. Integrated into the umbrella via `global.observability.enabled`.

**Tech Stack:** Helm v3, helm-unittest plugin, `kube-prometheus-stack` (prometheus-community), `loki` + `alloy` (grafana Helm repo), `monitoring.coreos.com/v1` CRDs (ServiceMonitor, PrometheusRule)

```yaml
# Stardog integration settings
stardog:
  metricsEnabled: true
  metricsPort: 5820
  metricsPath: /metrics
  # Namespace where Stardog runs. Empty = same namespace as this chart.
  namespace: ""
  serviceMonitor:
    interval: 30s
    scrapeTimeout: 10s
    labels: {}
  prometheusRule:
    enabled: true
    labels: {}
    # Threshold for high heap alert (0.0–1.0)
    heapUsageThreshold: "0.85"

# kube-prometheus-stack (upstream alias: prometheus)
# Full values reference: https://github.com/prometheus-community/helm-charts/tree/main/charts/kube-prometheus-stack
prometheus:
  fullnameOverride: prometheus
  grafana:
    fullnameOverride: grafana
    enabled: true
    adminPassword: prom-operator
    sidecar:
      dashboards:
        enabled: true
        label: grafana_dashboard
        labelValue: "1"
        searchNamespace: ALL
    additionalDataSources:
      - name: Loki
        type: loki
        url: http://loki:3100
        access: proxy
        isDefault: false
  prometheus:
    prometheusSpec:
      # Discover ALL ServiceMonitors in all namespaces (not just same-release ones)
      serviceMonitorSelectorNilUsesHelmValues: false
      serviceMonitorSelector: {}
      serviceMonitorNamespaceSelector: {}
  alertmanager:
    enabled: true
  nodeExporter:
    enabled: true
  kubeStateMetrics:
    enabled: true

# Loki — single-binary filesystem mode (suitable for dev/staging)
# For production, switch to distributed mode with object storage.
# Full values reference: https://github.com/grafana/loki/tree/main/production/helm/loki
loki:
  fullnameOverride: loki
  loki:
    auth_enabled: false
    commonConfig:
      replication_factor: 1
    storage:
      type: filesystem
    schemaConfig:
      configs:
        - from: "2024-01-01"
          store: tsdb
          object_store: filesystem
          schema: v13
          index:
            prefix: loki_index_
            period: 24h
  singleBinary:
    replicas: 1
  backend:
    replicas: 0
  read:
    replicas: 0
  write:
    replicas: 0

# Grafana Alloy — collects pod logs and ships to Loki
# Full values reference: https://github.com/grafana/alloy/tree/main/operations/helm/charts/alloy
alloy:
  fullnameOverride: alloy
  alloy:
    configMap:
      create: true
      content: |
        // Discover all pods in the cluster
        discovery.kubernetes "pods" {
          role = "pod"
        }

        // Relabel pod metadata into log stream labels
        discovery.relabel "pod_logs" {
          targets = discovery.kubernetes.pods.targets
          rule {
            source_labels = ["__meta_kubernetes_namespace"]
            target_label  = "namespace"
          }
          rule {
            source_labels = ["__meta_kubernetes_pod_name"]
            target_label  = "pod"
          }
          rule {
            source_labels = ["__meta_kubernetes_pod_container_name"]
            target_label  = "container"
          }
          rule {
            source_labels = ["__meta_kubernetes_pod_label_app_kubernetes_io_name"]
            target_label  = "app"
          }
        }

        // Collect logs from discovered pods
        loki.source.kubernetes "pods" {
          targets    = discovery.relabel.pod_logs.output
          forward_to = [loki.write.local.receiver]
        }

        // Write logs to Loki (uses fullnameOverride: loki → service name is predictable)
        loki.write "local" {
          endpoint {
            url = "http://loki:3100/loki/api/v1/push"
          }
        }
  rbac:
    create: true
  serviceAccount:
    create: true
  controller:
    type: daemonset
```

{{- if .Values.stardog.prometheusRule.enabled }}
apiVersion: monitoring.coreos.com/v1
kind: PrometheusRule
metadata:
  name: {{ include "sdcommon.fullname" . }}-stardog
  namespace: {{ .Release.Namespace }}
  labels:
    {{- include "observability.labels" . | nindent 4 }}
    {{- with .Values.stardog.prometheusRule.labels }}

    {{- end }}

  groups:

          for: 5m
          labels:
            severity: critical
          annotations:
            summary: "Stardog instance {{ "{{" }} $labels.instance {{ "}}" }} is down"
            description: "Stardog has been unreachable for more than 5 minutes."

        - alert: StardogHighHeapUsage
          expr: |
            stardog_jvm_heap_used_bytes / stardog_jvm_heap_max_bytes
            > {{ .Values.stardog.prometheusRule.heapUsageThreshold }}
          for: 10m
          labels:
            severity: warning
          annotations:
            summary: "Stardog heap usage is high on {{ "{{" }} $labels.instance {{ "}}" }}"
            description: >-
              Stardog JVM heap usage has exceeded
              {{ .Values.stardog.prometheusRule.heapUsageThreshold | float64 | mulf 100 | ceil }}%
              for more than 10 minutes.

        - alert: StardogQueryLatencyHigh
          expr: |
            rate(stardog_query_response_time_ms_sum[5m])
            / rate(stardog_query_response_time_ms_count[5m]) > 5000

            severity: warning
          annotations:

### Known Limitations (document in README when complete)
- **Loki storage**: Configured for single-binary filesystem mode. Not suitable for production multi-replica deployments. Switch to `loki.loki.storage.type: s3` with an object store for production.
- **Metric names**: The PrometheusRule and dashboard reference metric names (`stardog_jvm_heap_used_bytes`, `stardog_query_response_time_ms_*`) that must be validated against your exact Stardog version. Check `/metrics` endpoint on a running Stardog instance to verify names.
- **ServiceMonitor port name**: The ServiceMonitor targets port `http` on the Stardog service. Verify the Stardog chart's `service.yaml` names the HTTP port `http`, otherwise override with `stardog.serviceMonitor.portName`.
